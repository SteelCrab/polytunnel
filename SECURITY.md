# Security Policy

## Reporting a vulnerability

If you discover a security issue in this project, please report it privately before public disclosure.

1. Open a private security advisory on GitHub: `https://github.com/SteelCrab/polytunnel/security/advisories/new`
2. If private reporting is not available, create an issue with a clear `[Security]` prefix and include:
   - Affected version(s) and reproduction steps
   - Impact and attack scenario
   - Suggested mitigation (if any)

## Scope

- Polytunnel is currently at workspace version `0.1.0` (development phase).
- Security fixes are prioritized for immediate triage and patch release once confirmed.

## Communication

- We will acknowledge reports as soon as possible.
- If the issue is accepted, we provide:
  - Expected fix scope
  - Severity assessment
  - Timeline for verification and release

## Disclosure policy

- Keep report details private until a fix is available and a release path is confirmed.
- Do not submit credentials, secrets, or sensitive dataset content in public issues.
